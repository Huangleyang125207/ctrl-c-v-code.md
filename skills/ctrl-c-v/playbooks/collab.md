# Playbook: Collaboration

> Loaded by § 8.

## Principle

Structure your work so other engineers never encounter a question
your code doesn't already answer.

| You do this | So they never have to |
|---|---|
| PATTERN + decision comments | Ask "why?" |
| Match existing naming | Ask "why different?" |
| Scoped diffs | Ask "did you mean to change this?" |
| Clean commits | Ask "what happened?" |
| Self-reviewed code | Call you on a weekend |

The standard: colleague inherits your module, ships a feature,
never opens a message to you.
